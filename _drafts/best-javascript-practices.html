Do NOT use global variables

The variables here includes objects and functions. Since a web page might consist of many scripts, the global variables defined by a script can be overridden by other one.

Using shortcut notations

Using object literal instead of new Object():


var post = new Object();
post.title   = 'Best Javascript practices';
post.date    = new Date();
post.publish = function() {
    ...
};

// Better
var post = {
    title: '..',
    date: new Date(),
    publish: function() {
        ...
    }
};

Using [] for array:

var languages = new Array();
languages[0] = 'java';
languages[1] = 'javascript';
languages[2] = 'php';
languages[3] = 'swift';

// Better
var languages = ['java', 'javascript', 'php', 'swift'];

Using ternary notation:

var direction;
if (option === 'up') {
    direction = 1;
} else {
    direction = -1;
}

// Better
var direction= (option === 'up') ? 1 : -1;

One more popular usage is to provide the default value for variable:

if (err) {
    message = err;
} else {
    message = 'The input is not valid';
}

// Better
message = err || 'The input is not valid';
